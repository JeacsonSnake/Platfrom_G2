## 2. 接线验证步骤

### 2.1 准备工作

**工具清单**:
- 数字万用表（带通断测试和电压测量）
- 4.7KΩ电阻 × 4（每个传感器一个上拉电阻）
- 杜邦线若干
- USB-TTL串口线（用于ESP32调试）

**供电检查**:
```bash
# 测量ESP32-S3板载3.3V输出
- 红表笔 → 3.3V引脚
- 黑表笔 → GND
- 预期读数: 3.25V ~ 3.35V
```

### 2.2 逐步接线验证

#### 步骤1: 连接公共GND
```
验证点: 所有MAX31850的GND引脚 → ESP32的GND

万用表测试（通断档）:
- 从ESP32的GND测试到每个传感器的GND焊盘
- 应听到蜂鸣声（电阻<1Ω）
```

#### 步骤2: 连接公共VDD（3.3V）
```
验证点: 所有MAX31850的VDD引脚 → ESP32的3.3V

万用表测试（电压档）:
- 不连接传感器，先测量ESP32的3.3V输出是否正常
- 逐个连接传感器的VDD，测量电压不应明显下降
- 预期: 3.20V ~ 3.35V
```

#### 步骤3: 连接GPIO14到1-Wire总线
```
接线: ESP32的GPIO14 → 所有MAX31850的DQ引脚

重要: 每个传感器的DQ引脚都需要独立的上拉电阻到3.3V！

正确接法（星型拓扑）:
         3.3V
          │
    ┌─────┼─────┬─────┐
   4.7K  4.7K  4.7K  4.7K
    │     │     │     │
   DQ0   DQ1   DQ2   DQ3
    │     │     │     │
    └─────┴─────┴─────┘
              │
           GPIO14
              │
            (ESP32)

错误接法（多个传感器共用单个上拉）:
         3.3V
          │
        4.7K  ← 只有一个上拉，可能导致信号衰减
          │
    ┌─────┴─────┬─────┐
   DQ0   DQ1   DQ2   DQ3
```

#### 步骤4: 配置地址引脚（AD0, AD1）

使用万用表验证每个传感器的地址引脚电平:

| 传感器 | AD0目标 | AD1目标 | 万用表测量点 |
|--------|---------|---------|-------------|
| P1(U1) | GND(0V) | GND(0V) | AD0-GND≈0V, AD1-GND≈0V |
| P2(U2) | 3.3V | GND(0V) | AD0-GND≈3.3V, AD1-GND≈0V |
| P3(U3) | GND(0V) | 3.3V | AD0-GND≈0V, AD1-GND≈3.3V |
| P4(U4) | 3.3V | 3.3V | AD0-GND≈3.3V, AD1-GND≈3.3V |

### 2.3 1-Wire总线信号质量检查

#### 使用示波器检查（推荐）

```
示波器设置:
- 触发: 下降沿，1V电平
- 时基: 100μs/div
- 探头: 10×衰减，接在GPIO14和GND之间

预期波形:
1. Reset脉冲: 480μs低电平
   
   3.3V ─┐
        │
        └────────┐
                 │
   0V            └────────┐
        |←──480μs──→|

2. Presence响应: 在Reset后15-60μs内，从机拉低60-240μs
   
   3.3V ─┐
        │
        └────────┐
        ↑       │
   主机释放   从机拉低
                 │
   0V            └──┐
                    │
        |←15-60μs→|←60-240μs→|

3. 写1时隙: 低电平10μs
   
   3.3V ─┐   ┌──────────
        │   │
        └───┘
   0V     ↑
        10μs

4. 写0时隙: 低电平70μs
   
   3.3V ─┐         ┌───
        │         │
        └─────────┘
   0V     ↑
        70μs
```

#### 使用逻辑分析仪检查

```
逻辑分析仪设置:
- 采样率: 1MHz或更高
- 通道: CH0接GPIO14
- 触发: 下降沿

捕获数据后检查:
1. 第一个脉冲宽度约480μs（Reset）
2. 第二个脉冲（如有）宽度60-240μs（Presence）
3. 后续脉冲宽度10μs（写1）或70μs（写0）
```

### 3.3 故障诊断

#### 问题1: "No device present during search"

**可能原因**:
- 1-Wire总线开路（GPIO14未连接到传感器DQ）
- 上拉电阻缺失或阻值错误
- 传感器电源未连接
- GPIO14引脚损坏

**排查步骤**:
```
1. 测量GPIO14对地电压（空闲时应为3.3V）
   - 如果为0V: 检查是否有短路到GND
   - 如果为3.3V: 检查Reset时是否能看到下降沿

2. 测量DQ引脚对地电阻（断电时）
   - 应为4.7KΩ左右（上拉电阻值）
   - 如果为0Ω: 检查是否有短路
   - 如果为无穷大: 检查上拉电阻是否连接

3. 测量VDD引脚电压（上电时）
   - 应为3.3V
   - 如果为0V: 检查电源连接
```

#### 问题2: "1-Wire bus short to GND detected"

**可能原因**:
- DQ引脚短路到GND
- 上拉电阻焊接错误（接到了GND而非3.3V）
- 传感器损坏

**排查步骤**:
```
1. 断电，测量GPIO14对地电阻
   - 应约为4.7KΩ（上拉电阻值）
   - 如果接近0Ω: 存在短路

2. 逐个断开传感器，找出短路的设备

3. 检查PCB是否有焊锡桥连
```

#### 问题3: "ROM ID CRC error"

**可能原因**:
- 信号完整性问题（线路太长、干扰）
- 上拉电阻阻值不合适（太大或太小）
- 总线电容过大

**解决方案**:
```
1. 缩短1-Wire总线长度（建议<10米）

2. 调整上拉电阻值:
   - 短距离(<1米): 4.7KΩ
   - 中等距离(1-5米): 2.2KΩ
   - 长距离(5-10米): 1KΩ

3. 降低总线速度（当前实现已是标准速度）

4. 检查电源稳定性，增加去耦电容
```

#### 问题4: 只找到部分传感器

**可能原因**:
- 某个传感器接线错误
- 地址引脚配置错误（两个传感器地址冲突）
- 传感器损坏

**排查步骤**:
```
1. 单独测试每个传感器:
   - 只连接一个传感器，确认能被识别
   - 逐个增加传感器，找出问题设备

2. 检查地址引脚连接:
   - 用万用表测量AD0和AD1的电平
   - 确认与目标地址配置一致

3. 检查是否有重复地址:
   - 两个传感器不能有相同的AD0/AD1组合
```

#### 问题5: "Thermocouple open circuit"

**可能原因**:
- K型热电偶未连接
- 热电偶接线松动
- MAX31850的T+/T-引脚焊接问题

**排查步骤**:
```
1. 检查热电偶连接:
   - 确认热电偶插头插入到位
   - 测量热电偶电阻（应<100Ω，取决于长度）

2. 使用max31850_dump_scratchpad()查看详细数据:
   I (1234) MAX31850: Sensor 0 Scratchpad Dump:
   I (1235) MAX31850:   [0] Temp LSB: 0x00
   I (1236) MAX31850:   [1] Temp MSB: 0x00
   I (1237) MAX31850:   [2] Reserved: 0x00
   I (1238) MAX31850:   [3] Config:   0x00
   I (1239) MAX31850:   [4] Fault:    0x01  ← 0x01表示OC故障
   ...

3. 检查T+/T-引脚是否有虚焊
```

## 5. 常见问题FAQ

### Q1: 为什么要用4个独立的上拉电阻？

A: 虽然1-Wire总线可以共用单个上拉电阻，但在多设备并联时：
1. 独立上拉提供更好的信号完整性
2. 如果某个传感器损坏导致总线故障，不影响其他传感器
3. 方便单独调试和更换传感器

### Q2: 可以用GPIO引脚内部上拉代替外部4.7KΩ电阻吗？

A: **不推荐**。ESP32内部上拉电阻通常为30-50KΩ，对于1-Wire协议来说太弱，无法提供足够的电流驱动总线。

### Q3: MAX31850可以工作在5V吗？

A: 不可以。MAX31850KATB+的工作电压范围是3.0V-3.6V，必须使用3.3V供电。

### Q4: 热电偶最长可以接多长？

A: 取决于导线类型和干扰环境：
- 普通K型热电偶线：建议<10米
- 屏蔽双绞线：可达30米
- 使用补偿导线延长

### Q5: 如何提高温度读取精度？

A: 1. 确保MAX31850的电源稳定，增加100nF去耦电容
   2. 避免热电偶导线与其他电源线平行布置
   3. 使用屏蔽线减少EMI干扰
   4. 定期校准（MAX31850内置冷端补偿）
