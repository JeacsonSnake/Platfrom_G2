# ESP32S3 Motor Control PCNT Noise Filter and Diagnostic Enhancement

**日期**: 2026-03-20  
**分支**: `feature/motor-control-config`  
**任务描述**: 修复PCNT在未通电时的噪声计数问题，添加Motor 3 PCNT诊断功能

---

## 1. 背景

### 历史问题回顾
根据 2026-03-20 15:45 的测试记录，ESP32S3 电机控制存在以下问题：

**问题①**: 最开始的日志中有多行类似于：
```
[2026-03-20 15:45:10.285] I (12693) PCNT_EVENT: Motor 1 idle, PCNT=2920
[2026-03-20 15:45:10.287] I (12703) PWM_EVENT: PWM channel 1 duty set to 8191.
```

**现象**: 
- USB供电（5V）但没有12V供电时，Motor 1和Motor 2的PCNT有大量异常计数
- 即使是接通了从控的12V外部供电也依然存在
- 先接通12V外部供电，再通过USB连接ESP32S3，则无该问题

**问题②**: P9接口所连接的"Motor 3"的PCNT_EVENT未侦测出速度
- 实际输出速度并非预期速度
- PWM输出正常（duty从7589降到0），说明软件控制正常
- PCNT始终为0，说明编码器信号没有到达GPIO 9

## 2. 问题分析与修复方案

### 2.1 问题①：未通电时PCNT噪声计数

#### 问题分析
| 项目 | 问题描述 |
|------|----------|
| **根因** | 12V未供电时，电机驱动板未上电，编码器信号线（GPIO 2, 5, 7, 9）处于浮动状态 |
| **现象** | 浮动GPIO接收到噪声信号，导致PCNT误计数（如266, 168, 6570, 6647等） |
| **影响** | 电机未运行时，PCNT显示异常计数，干扰PID控制 |

#### 修复方案
在 `main/pcnt.c` 中增强噪声过滤机制：

1. **添加系统启动保护期**（3秒）：
   - 系统启动后的前3秒内，如果电机速度为0，强制清零PCNT计数
   - 等待12V电源稳定后再开始正常监测

2. **增强异常值检测**：
   - 启动保护期结束后，正常启用异常值检测
   - 检测到超过阈值的计数时，记录警告日志并重置为0

3. **空闲状态噪声过滤**：
   - 电机停止时，如果PCNT超过50，视为噪声并清零

```c
// 系统启动时间戳，用于启动保护期
static uint32_t system_boot_time = 0;

// 启动保护期：3秒（等待12V电源稳定）
const uint32_t STARTUP_PROTECTION_MS = 3000;
// 空闲时的噪声阈值：电机停止时，PCNT超过此值视为噪声
const int IDLE_NOISE_THRESHOLD = 50;
```

### 2.2 问题②：P9接口（Motor 3 / PH2.0-LI-5P_004）PCNT始终为0

#### 问题背景
用户反馈：
- 将Motor 3线路临时接到P8（PH2.0-LI-5P_003）以"Motor 2"身份运行正常
- 说明**物理电机Motor 3本身正常**
- 但P9接口（对应软件Motor 3，PWM GPIO8, FG GPIO9）始终读不到PCNT信号

#### 问题分析
| 项目 | 分析结果 |
|------|----------|
| **P8接口测试** | Motor 3通过P8运行正常，排除电机本身故障 |
| **P9接口定义** | PWM=GPIO8, FG=GPIO9（见电路图PH2.0-LI-5P_004.png） |
| **GPIO 9** | ESP32-S3通用GPIO，非USB专用引脚 |
| **可能原因** | 1. P9接口硬件连接问题<br>2. FG信号线（IO9）断开或接触不良<br>3. P9接口电路板线路问题 |

#### 修复方案
添加 **P9接口专项诊断功能**：

```c
// P9接口（Motor 3）专项诊断任务
void p9_interface_diagnostic_task(void* params)
{
    // 配置GPIO 9为输入模式，直接读取电平
    // 监测5秒内GPIO 9电平变化次数
    // 输出诊断结果：
    // - 变化次数=0：无信号，检查硬件连接
    // - 变化次数<10：信号弱，可能接触不良
    // - 变化次数正常：GPIO 9工作正常
}
```

诊断任务在系统启动时自动运行，帮助定位P9接口问题。

## 3. 实现方案

### 3.1 修改文件

#### `main/pcnt.c`

**添加系统启动保护**：
```c
// 系统启动时间戳，用于启动保护期
static uint32_t system_boot_time = 0;

void pcnt_func_init()
{
    // 记录系统启动时间
    system_boot_time = esp_timer_get_time() / 1000; // 转换为毫秒
    // ... 其余初始化代码
}
```

**增强噪声过滤和诊断**：
```c
void pcnt_monitor(void* params)
{
    // ... 原有变量
    bool startup_protection_active = true;
    const uint32_t STARTUP_PROTECTION_MS = 3000;
    const int IDLE_NOISE_THRESHOLD = 50;
    
    // PCNT 运行统计，用于诊断
    static uint32_t pcnt_zero_count[4] = {0, 0, 0, 0};
    static uint32_t pcnt_total_samples[4] = {0, 0, 0, 0};
    
    while(1)
    {
        // 检查启动保护期是否结束
        uint32_t current_time = esp_timer_get_time() / 1000;
        if (startup_protection_active && (current_time - system_boot_time > STARTUP_PROTECTION_MS)) {
            startup_protection_active = false;
            ESP_LOGI(TAG, "Motor %d startup protection ended", index);
        }
        
        // 获取PCNT计数
        pcnt_unit_get_count(unit, &pcnt_count_list[index]);
        pcnt_unit_clear_count(unit);
        
        // 统计PCNT采样数据（用于诊断Motor 3问题）
        pcnt_total_samples[index]++;
        if (pcnt_count_list[index] == 0) {
            pcnt_zero_count[index]++;
        }
        
        // 每50个样本（约10秒）输出一次诊断信息
        if (pcnt_total_samples[index] % 50 == 0 && motor_speed_list[index] != 0) {
            uint32_t zero_rate = (pcnt_zero_count[index] * 100) / pcnt_total_samples[index];
            if (zero_rate > 80) {
                ESP_LOGW(TAG, "Motor %d PCNT诊断: %d%%采样为0，可能存在硬件连接问题", 
                         index, zero_rate);
            }
        }
        
        // 启动保护期特殊处理
        if (startup_protection_active && motor_speed_list[index] == 0) {
            if (pcnt_count_list[index] > 0) {
                pcnt_count_list[index] = 0; // 强制清零噪声
            }
        }
        
        // 空闲状态噪声过滤
        if (motor_speed_list[index] == 0 && idle == false && pcnt_count_list[index] > IDLE_NOISE_THRESHOLD) {
            ESP_LOGW(TAG, "Motor %d idle noise detected: PCNT=%d, filtering", index, pcnt_count_list[index]);
            pcnt_count_list[index] = 0;
        }
        
        // ... 其余处理逻辑
    }
}
```

## 4. 修复验证建议

### 4.1 问题①验证

**测试步骤**：
1. 仅USB供电（不接通12V），启动ESP32
2. 观察日志，确认启动保护期内（前3秒）PCNT计数被清零
3. 接通12V电源，发送电机指令，确认PCNT正常计数
4. 停止电机，确认空闲时噪声被过滤

**预期结果**：
- 启动保护期内：`startup protection: filtering PCNT noise`
- 正常工作时：`Motor X running, PCNT=XXX/s`
- 空闲时：`Motor X idle, PCNT=0`（无异常大值）

### 4.2 问题②验证

**测试步骤**：
1. 启动ESP32，等待系统初始化完成
2. 发送Motor 3指令（如 `cmd_3_75_5`）
3. 观察日志中的PCNT诊断信息

**可能结果**：
- **情况A**: PCNT正常计数 → 问题已解决
- **情况B**: 诊断日志显示 `Motor 3 PCNT诊断: 100%采样为0` → 硬件连接问题

**如果问题B持续，建议检查**：
1. P9接口的FG信号线是否确实连接到GPIO 9
2. 使用万用表检查GPIO 9引脚是否有脉冲信号
3. 检查PCNT单元3是否正确初始化（查看初始化日志）
4. 尝试更换Motor 3的电机或编码器

## 9. 验证结果与最终结论

### 9.1 测试记录（2026-03-25 13:36）

**测试环境**：
- P9接口（PH2.0-LI-5P_004）连接Motor 3
- P8接口（PH2.0-LI-5P_003）作为对比测试

#### P9接口专项诊断结果

```
[13:36:34.741] P9接口（Motor 3）专项诊断启动
[13:36:34.746] GPIO: PWM=GPIO8, FG=GPIO9
[13:36:34.749] 物理接口: PH2.0-LI-5P_004
[13:36:34.768] P9诊断: GPIO 9 初始电平=1
[13:36:34.866] P9诊断: GPIO 9 电平变化 1->0 (变化次数:1)
[13:36:39.773] P9接口诊断结果:
   - GPIO 9 电平变化次数: 1
⚠️ 信号较弱，可能接触不良
```

**诊断结论**：GPIO 9在5秒监测期内仅有1次电平变化，正常情况下应有数百次变化。

#### Motor 3（P9接口）运行测试

**指令**：`cmd_3_75_7`

| 项目 | 结果 |
|------|------|
| PWM输出 | ✅ 正常（duty 7589→0） |
| PCNT读数 | ❌ **始终为0** |
| 诊断警告 | `Motor 3 PCNT诊断: 99%采样为0` |

**现象**：PID控制输出不断增加（pid_out=602→7937），但电机实际转速无法反馈。

#### Motor 2（P8接口）对比测试

**指令**：`cmd_2_75_7`

| 项目 | 结果 |
|------|------|
| PWM输出 | ✅ 正常 |
| PCNT读数 | ✅ 正常计数（0→10→55→70→75...） |
| 控制效果 | ✅ 稳定在75/s |

### 9.2 问题根因确认

| 对比项 | P8 (Motor 2) | P9 (Motor 3) | 结论 |
|--------|--------------|--------------|------|
| PWM控制 | 正常 | 正常 | 软件无问题 |
| FG信号 | 正常 | **无信号** | **P9硬件故障** |
| 物理电机 | - | - | 电机本身正常 |

### 9.3 最终结论

**问题定位**：**P9接口（PH2.0-LI-5P_004）的FG信号线（IO9）存在硬件故障**

**软件层面**：
- ✅ PCNT噪声过滤功能正常
- ✅ 启动保护期功能正常
- ✅ P9接口专项诊断功能正常
- ✅ 问题定位为硬件故障

**硬件层面问题**：
1. 接口插头接触不良
2. FG信号线（IO9）断路
3. PCB走线问题
4. 焊点虚焊

**建议修复措施**：
1. 检查P9接口的物理插头是否插紧
2. 使用万用表测量P9接口第5脚（FG）到GPIO 9的通路
3. 检查PCB上IO9走线是否有断裂
4. 检查焊点是否虚焊

**临时解决方案**：
- 继续使用P8接口控制Motor 3（当前做法）
