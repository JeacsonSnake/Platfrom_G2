## 1. 数据手册关键要求

### 1.1 读时隙时序要求 (Page 20)

**原文摘录**:
> "A read time slot is initiated by the master device pulling the 1-Wire bus low for a **minimum of 1μs (tINIT)** and then releasing the bus."

> "Output data from the slave is valid for 15μs after the falling edge that initiated the read time slot. Therefore, **the master must release the bus and then sample the bus state within 15μs from the start of the slot.**"

**关键参数**:
| 参数 | 符号 | 要求 |
|------|------|------|
| 初始化拉低时间 | tINIT | ≥ 1μs |
| 采样时间窗口 | tRDV | < 15μs (从下降沿开始) |
| 时隙总时长 | tSLOT | 60-120μs |

## 3. 风险评估

### 3.1 方案1风险评估

| 风险项 | 等级 | 说明 |
|--------|------|------|
| tINIT = 1μs (临界值) | ⚠️ **中** | 正好在数据手册下限，考虑到实际延时可能略长/略短 |
| 采样点 11μs | ✅ **低** | 距离15μs上限有4μs余量 |
| 总时隙 66μs | ✅ **低** | 完全合规 |

**潜在问题**:
- `esp_rom_delay_us(1)` 的实际延时可能因中断或系统负载略有偏差
- 如果实际延时只有 0.9μs，则违反数据手册要求

## 5. 结论与建议

### 5.1 验证结论

| 方案 | 合规性 | 安全余量 | 推荐度 |
|------|--------|----------|--------|
| **方案1** | ✅ 完全符合 | 4μs | ⭐⭐⭐⭐ |
| **方案2** | ✅ 完全符合 | 5μs | ⭐⭐⭐⭐⭐ |

### 5.2 推荐决策

**建议使用方案2**，理由：

1. **更高的安全性**: tINIT = 2μs 比 1μs 更可靠，避免临界值风险
2. **充足的安全余量**: 5μs 余量比 4μs 更保险
3. **仍能有效解决问题**: 10μs 采样点同样能让从设备有足够时间驱动总线
4. **符合数据手册精神**: 数据手册建议 "keeping tINIT as short as possible"，2μs 已经足够短

### 5.3 实施参数 (方案2)

```c
// 读时序参数优化 - 方案2 (推荐)
#define ONE_WIRE_READ_LOW           2       // 2μs，尽快释放总线 (有1μs安全余量)
#define ONE_WIRE_READ_SAMPLE        8       // 8μs，延后采样给从设备时间
#define ONE_WIRE_READ_HIGH          56      // 56μs，调整恢复时间
// 读时隙总时间：2 + 8 + 56 = 66μs (>60μs)
// 采样点：10μs (<15μs，距离上限有5μs余量)
```
