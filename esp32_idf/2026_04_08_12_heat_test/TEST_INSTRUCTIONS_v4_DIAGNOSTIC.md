# MAX31850 诊断测试说明 v4 (5μs额外恢复时间)

**日期**: 2026-04-08  
**分支**: `feature/heating`  
**提交**: `83a400f`  
**版本**: 诊断测试 - 5μs额外恢复时间

---

## 1. 测试目的

验证位反转问题（家族码0x3B读成0xDC）是否由**信号上升沿过慢**导致。

### 背景
- 4.7KΩ上拉电阻导致信号上升沿慢
- 之前的时序调整（采样点13μs）仍无法解决问题
- 需要确认问题根源是时序还是其他因素

---

## 2. 修改内容

### 2.1 时序参数

| 参数 | 数值 | 说明 |
|------|------|------|
| **ONE_WIRE_READ_LOW** | 1μs | 拉低时间 |
| **额外恢复时间** | **5μs** | **新增：给上拉电阻建立信号** |
| **ONE_WIRE_READ_SAMPLE** | 12μs | 采样延迟 |
| **ONE_WIRE_READ_HIGH** | 52μs | 恢复时间 |
| **实际采样点** | **18μs** | **1 + 5 + 12 = 18μs** |

### 2.2 代码修改

```c
// 释放总线
gpio_set_level_fast(g_driver.gpio_num, 1);

// 额外恢复时间 - 给弱上拉(4.7KΩ)更多时间建立信号
esp_rom_delay_us(5);

esp_rom_delay_us(ONE_WIRE_READ_SAMPLE);
```

---

## 3. 预期结果分析

### 场景A: 如果测试成功 ✅

```
ROM bits: 00111011...          <-- 家族码正确 (0x3B)
CRC check: PASS
Found 4 sensors on GPIO14
```

**结论**: 
- 位反转问题**确实由信号上升沿过慢导致**
- 4.7KΩ上拉电阻是根本原因
- **必须更换上拉电阻**为2.2KΩ或1KΩ才能符合数据手册

### 场景B: 如果测试失败 ❌

```
ROM bits: 11011100...          <-- 仍是0xDC
CRC check: FAIL
Found 0 sensors on GPIO14
```

**结论**:
- 位反转问题**不是由上升沿时间导致**
- 可能存在其他问题：
  - GPIO配置问题
  - 传感器硬件故障
  - 代码逻辑错误
  - 电气干扰

## 5. 结果判定

| 结果 | ROM家族码 | 含义 | 下一步行动 |
|------|-----------|------|------------|
| **成功** | 0x3B (00111011) | 上升沿是问题根源 | 更换上拉电阻为2.2KΩ |
| **失败** | 0xDC (11011100) | 上升沿不是问题 | 检查其他硬件/代码问题 |

## 6. 注意事项

⚠️ **重要**: 此修改超出MAX31850数据手册要求
- 数据手册要求：采样点 < 15μs
- 此修改采样点：18μs
- **仅用于诊断，不建议作为最终方案**
