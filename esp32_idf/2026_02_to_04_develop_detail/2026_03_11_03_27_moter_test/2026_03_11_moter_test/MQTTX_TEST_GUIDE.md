# MQTTX 测试指南 - CHB-BLDC2418 电机控制

**工具**: MQTTX (图形化 MQTT 客户端)  
**Broker**: EMQX (192.168.110.31:1883)  
**替代**: 本指南替代 mosquitto 命令行工具  

---

## 1. MQTTX 连接配置

### 1.1 创建新连接

打开 MQTTX → 点击 "+ 新建连接"

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **名称** | ESP32_Motor_Test | 任意名称 |
| **Client ID** | MQTTX_Test_xxx | 自动生成或自定义 |
| **服务器地址** | 192.168.110.31 | EMQX Broker IP |
| **端口** | 1883 | MQTT 默认端口 |
| **协议** | MQTT | 默认 |
| **Path** | /mqtt | WebSocket 时用，普通 MQTT 可留空 |

### 1.2 认证配置 (如有需要)

根据 AGENTS.md，MQTT 配置了用户名密码：

| 配置项 | 值 |
|--------|-----|
| **用户名** | ESP32_1 |
| **密码** | 123456 |

⚠️ **注意**: 如果 ESP32 端的 MQTT 配置了认证，测试客户端也需要配置相同的凭据才能发布到控制主题。

### 1.3 连接测试

点击 **"连接"** 按钮：
- ✅ 右上角显示绿色圆点 = 连接成功
- ❌ 红色 = 检查网络和配置

---

## 2. 订阅数据主题 (接收电机反馈)

### 2.1 订阅 esp32_1/data

在 MQTTX 中：
1. 连接成功后，点击 "+ 新建订阅"
2. 主题名称: `esp32_1/data`
3. QoS: `0` (或 `1`)
4. 点击 "确认"

### 2.2 预期接收的消息类型

| 消息格式 | 含义 | 示例 |
|----------|------|------|
| `pcnt_count_X_YYY` | 电机X的转速计数 | `pcnt_count_0_228` |
| `pwm_set_X_YYY` | PWM占空比更新 | `pwm_set_0_4000` |

---

## 3. 发布控制命令 (控制电机)

### 3.1 发布到 esp32_1/control

在 MQTTX 中发送消息：

1. 点击 "新建发布"
2. 主题: `esp32_1/control`
3. QoS: `1` (推荐，确保命令到达)
4. 输入消息内容

### 3.2 命令格式

```
cmd_<电机编号>_<速度>_<持续时间>
```

| 参数 | 范围 | 说明 |
|------|------|------|
| 电机编号 | 0~3 | 对应 Motor 0~3 |
| 速度 | 0~450 | 0=停止, 450=满速(4500RPM) |
| 持续时间 | 1~65535 | 运行秒数 |

### 3.3 常用测试命令

#### 基础测试

| 测试目的 | 命令 | 预期结果 |
|----------|------|----------|
| Motor 0 半速5秒 | `cmd_0_225_5` | 电机0以50%速度转5秒后停止 |
| Motor 0 全速3秒 | `cmd_0_450_3` | 电机0以100%速度转3秒后停止 |
| Motor 0 停止 | `cmd_0_0_1` | 电机0立即停止 |

#### 多电机并行测试

发送4条命令（在4个发布窗口或快速连续发送）：

```
cmd_0_225_10
cmd_1_225_10
cmd_2_225_10
cmd_3_225_10
```

**预期**: 4个电机同时以50%速度运行10秒

#### 速度梯度测试

依次发送：

```
cmd_0_0_3    (0%, 停止3秒)
cmd_0_112_3  (25%, 3秒)
cmd_0_225_3  (50%, 3秒)
cmd_0_337_3  (75%, 3秒)
cmd_0_450_3  (100%, 3秒)
cmd_0_0_3    (停止)
```

## 4. 实际测试步骤 (Step by Step)

### 步骤 1: 准备工作

- [ ] ESP32 已烧录固件并启动
- [ ] 电机接线正确 (至少 Motor 0)
- [ ] MQTTX 连接到 192.168.110.31:1883
- [ ] 订阅 `esp32_1/data` 主题

### 步骤 2: 验证心跳 (确认连接)

观察 `esp32_1/data` 是否定期收到心跳消息：
```
heartbeat
```

**✓ 通过**: 每30秒收到一次心跳

### 步骤 3: 单电机启动测试

1. **发送**: `cmd_0_225_5`
2. **观察**: 
   - 电机是否开始旋转？
   - `esp32_1/data` 是否收到 `pcnt_count_0_XXX`？
3. **5秒后**: 电机是否自动停止？

**✓ 通过标准**: 
- 电机平滑旋转
- 收到 pcnt 数据且数值接近 225
- 5秒后自动停止

### 步骤 4: 速度准确性测试

| 发送命令 | 预期 PCNT | 容差 | 实际 PCNT | 结果 |
|----------|-----------|------|-----------|------|
| `cmd_0_112_5` | ~112 | ±10% | _____ | □ |
| `cmd_0_225_5` | ~225 | ±10% | _____ | □ |
| `cmd_0_450_5` | ~450 | ±10% | _____ | □ |

### 步骤 5: 多电机测试 (如有多个电机)

依次发送：
```
cmd_0_225_5
cmd_1_225_5
cmd_2_225_5
cmd_3_225_5
```

**✓ 通过标准**: 所有电机同时旋转，各自返回 pcnt 数据

## 5. 高级测试技巧

### 5.1 使用 MQTTX 的脚本功能 (Script)

MQTTX 支持自定义脚本处理消息。可以编写简单的脚本来解析 pcnt 数据：

```javascript
// 示例：提取 pcnt 数值并计算误差
function handlePayload(payload) {
  if (payload.includes('pcnt_count')) {
    const parts = payload.split('_');
    const motor = parts[2];
    const value = parseInt(parts[3]);
    console.log(`Motor ${motor}: PCNT = ${value}`);
    // 可以在这里添加误差计算逻辑
  }
  return payload;
}

execute(handlePayload);
```

### 5.2 保存测试配置

将常用命令保存为 MQTTX 的 "收藏"：
1. 发送一条命令后，点击消息旁的 ⭐ 收藏
2. 命名收藏: "Motor0_HalfSpeed"
3. 后续可直接从收藏夹选择发送

### 5.3 使用多窗口监控

打开多个 MQTTX 窗口：
- **窗口1**: 订阅 `esp32_1/data` (监控反馈)
- **窗口2**: 发布到 `esp32_1/control` (发送命令)
- **窗口3**: 订阅 `esp32_1/heartbeat` (监控心跳)

## 6. 故障排查

### Q: MQTTX 无法连接
| 检查项 | 解决方案 |
|--------|----------|
| 网络连通性 | `ping 192.168.110.31` |
| 端口开放 | `telnet 192.168.110.31 1883` |
| 防火墙 | 关闭防火墙或开放 1883 端口 |

### Q: 发送命令后电机无反应
| 检查项 | 排查方法 |
|--------|----------|
| 主题正确性 | 确认是 `esp32_1/control` |
| 命令格式 | 确认格式为 `cmd_0_225_5` |
| QoS 级别 | 尝试 QoS 0 和 QoS 1 |
| ESP32 状态 | 查看 ESP32 串口日志 |

### Q: 能控制但无数据返回
| 检查项 | 排查方法 |
|--------|----------|
| 订阅主题 | 确认订阅了 `esp32_1/data` |
| FG 接线 | 检查 GPIO2/5/7/9 接线 |
| 电机旋转 | 确认电机实际在转 |
