### 1.3 ESP32-S3 GPIO14 连接确认

根据原理图 **ESP32-S3-DevKitC-1-master.pdf** / **ESP32-S3-DevKitC-1-slave.pdf**：

```
ESP32-S3 GPIO14 (IO14)
    │
    ├──→ 连接到 MAX31850 的 DQ 引脚（所有4个传感器并联）
    │
    └──→ 每个传感器的 DQ 引脚通过 4.7KΩ 电阻上拉到 3.3V
```

**重要**: GPIO14 必须配置为 **开漏输出 (Open-Drain)** 模式，配合外部上拉电阻工作。

## 2. 接线验证步骤

### 2.1 准备工作

**工具清单**:
- 数字万用表（电压档 + 通断档）
- 4.7KΩ电阻 × 4（每个传感器一个）
- 杜邦线若干

**供电检查**:
```bash
# 测量ESP32-S3板载3.3V输出
- 红表笔 → 3.3V引脚
- 黑表笔 → GND
- 预期读数: 3.25V ~ 3.35V
```

### 2.2 逐步验证

#### 步骤1: 连接公共GND
```
验证点: 所有MAX31850的GND引脚 → ESP32的GND

万用表测试（通断档）:
- 从ESP32的GND测试到每个传感器的GND焊盘
- 应听到蜂鸣声（电阻<1Ω）
```

#### 步骤2: 连接公共VDD（3.3V）
```
验证点: 所有MAX31850的VDD引脚 → ESP32的3.3V

万用表测试（电压档）:
- 不连接传感器，先测量ESP32的3.3V输出是否正常
- 逐个连接传感器的VDD，测量电压不应明显下降
- 预期: 3.20V ~ 3.35V
```

#### 步骤3: 连接GPIO14到1-Wire总线
```
接线: ESP32的GPIO14 → 所有MAX31850的DQ引脚

重要: 每个传感器的DQ引脚都需要独立的上拉电阻到3.3V！

正确接法（星型拓扑）:
         3.3V
          │
    ┌─────┼─────┬─────┐
   4.7K  4.7K  4.7K  4.7K
    │     │     │     │
   DQ0   DQ1   DQ2   DQ3
    │     │     │     │
    └─────┴─────┴─────┘
              │
           GPIO14
              │
            (ESP32)

错误接法（多个传感器共用单个上拉）:
         3.3V
          │
        4.7K  ← 只有一个上拉，可能导致信号衰减
          │
    ┌─────┴─────┬─────┐
   DQ0   DQ1   DQ2   DQ3
```

#### 步骤4: 配置地址引脚（AD0, AD1）

使用万用表验证每个传感器的地址引脚电平:

| 传感器 | AD0目标 | AD1目标 | 万用表测量点 |
|--------|---------|---------|-------------|
| P1(U1) | GND(0V) | GND(0V) | AD0-GND≈0V, AD1-GND≈0V |
| P2(U2) | 3.3V | GND(0V) | AD0-GND≈3.3V, AD1-GND≈0V |
| P3(U3) | GND(0V) | 3.3V | AD0-GND≈0V, AD1-GND≈3.3V |
| P4(U4) | 3.3V | 3.3V | AD0-GND≈3.3V, AD1-GND≈3.3V |

## 4. 故障诊断

### 4.1 "No device present during search"

**可能原因**:
- 1-Wire总线开路（GPIO14未连接到传感器DQ）
- 上拉电阻缺失或阻值错误
- 传感器电源未连接
- GPIO14引脚损坏

**排查步骤**:
```
1. 测量GPIO14对地电压（空闲时应为3.3V）
   - 如果为0V: 检查是否有短路到GND
   - 如果为3.3V: 检查Reset时是否能看到下降沿

2. 测量DQ引脚对地电阻（断电时）
   - 应为4.7KΩ左右（上拉电阻值）
   - 如果为0Ω: 检查是否有短路
   - 如果为无穷大: 检查上拉电阻是否连接

3. 测量VDD引脚电压（上电时）
   - 应为3.3V
   - 如果为0V: 检查电源连接
```

### 4.2 "ROM ID CRC error"

**可能原因**:
- 信号完整性问题（线路太长、干扰）
- 上拉电阻阻值不合适（太大或太小）
- 总线电容过大

**解决方案**:
```
1. 缩短1-Wire总线长度（建议<10米）

2. 调整上拉电阻值:
   - 短距离(<1米): 4.7KΩ
   - 中等距离(1-5米): 2.2KΩ
   - 长距离(5-10米): 1KΩ

3. 降低总线速度（当前实现已是标准速度）

4. 检查电源稳定性，增加去耦电容
```

### 4.3 "Thermocouple open circuit"

**可能原因**:
- 热电偶未连接
- 热电偶接线松动
- MAX31850的T+/T-引脚焊接问题

**排查步骤**:
```
1. 检查热电偶连接:
   - 确认热电偶插头插入到位
   - 测量热电偶电阻（应<100Ω，取决于长度）

2. 使用max31850_dump_scratchpad()查看详细数据:
   I (1234) MAX31850: Sensor 0 Scratchpad Dump:
   I (1235) MAX31850:   [0] Temp LSB: 0x00
   I (1236) MAX31850:   [1] Temp MSB: 0x00
   I (1237) MAX31850:   [2] Reserved: 0x00
   I (1238) MAX31850:   [3] Config:   0x00
   I (1239) MAX31850:   [4] Fault:    0x01  ← 0x01表示OC故障
   ...

3. 检查T+/T-引脚是否有虚焊
```

### 4.4 只找到部分传感器

**可能原因**:
- 某个传感器接线错误
- 地址引脚配置错误（两个传感器地址冲突）
- 传感器损坏

**排查步骤**:
```
1. 单独测试每个传感器:
   - 只连接一个传感器，确认能被识别
   - 逐个增加传感器，找出问题设备

2. 检查地址引脚连接:
   - 用万用表测量AD0和AD1的电平
   - 确认与目标地址配置一致

3. 检查是否有重复地址:
   - 两个传感器不能有相同的AD0/AD1组合
```

## 5. 技术要点

### 5.1 RMT vs GPIO Bit-bang

本驱动采用 **GPIO Bit-bang + 临界区保护** 实现1-Wire协议，原因：

| 方案 | 优点 | 缺点 |
|------|------|------|
| GPIO Bit-bang | 时序精确可控，支持读时隙的精确采样 | 占用CPU时间 |
| RMT TX | 硬件发送，不占用CPU | 需要配合RX接收，复杂度高 |
| RMT TX+RX | 全自动收发 | 1-Wire协议需要主机主动控制采样时机，RMT RX不适合 |

**实际实现**: 使用临界区保护（portENTER_CRITICAL）确保微秒级时序不被中断，配合精确的延迟函数。

### 5.2 开漏输出模式

GPIO14 必须配置为 **GPIO_MODE_INPUT_OUTPUT_OD**（开漏输出）:

```c
gpio_config_t io_conf = {
    .pin_bit_mask = (1ULL << onewire_pin),
    .mode = GPIO_MODE_INPUT_OUTPUT_OD,  // 开漏模式
    .pull_up_en = GPIO_PULLUP_ENABLE,   // 内部上拉（辅助外部4.7KΩ）
    .pull_down_en = GPIO_PULLDOWN_DISABLE,
    .intr_type = GPIO_INTR_DISABLE,
};
```

**原因**: ESP32-S3的GPIO14在推挽模式下无法正常驱动高电平，必须使用开漏模式配合外部上拉电阻。

### 5.3 1-Wire时序参数

| 参数 | 标准值 | 实现值 | 说明 |
|------|--------|--------|------|
| Reset低电平 | 480μs | 480μs | 复位脉冲 |
| Presence等待 | 70μs | 70μs | 等待设备响应 |
| Write 1低电平 | 1-15μs | 6μs | 写1时隙 |
| Write 0低电平 | 60-120μs | 60μs | 写0时隙 |
| Read采样点 | 15μs | 15μs | 释放后15μs采样 |

## 6. 常见问题FAQ

### Q1: 为什么要用4个独立的上拉电阻？

A: 虽然1-Wire总线可以共用单个上拉电阻，但在多设备并联时：
1. 独立上拉提供更好的信号完整性
2. 如果某个传感器损坏导致总线故障，不影响其他传感器
3. 方便单独调试和更换传感器

### Q2: 可以用GPIO引脚内部上拉代替外部4.7KΩ电阻吗？

A: **不推荐**。ESP32内部上拉电阻通常为30-50KΩ，对于1-Wire协议来说太弱，无法提供足够的电流驱动总线。

### Q3: MAX31850可以工作在5V吗？

A: 不可以。MAX31850KATB+的工作电压范围是3.0V-3.6V，必须使用3.3V供电。

### Q4: 如何提高温度读取精度？

A: 1. 确保MAX31850的电源稳定，增加100nF去耦电容
   2. 避免热电偶导线与其他电源线平行布置
   3. 使用屏蔽线减少EMI干扰
   4. 定期校准（MAX31850内置冷端补偿）
