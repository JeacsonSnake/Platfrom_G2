## 1. Background

### Hardware Requirements
Based on schematic analysis from `hardware_info/ESP32-S3-DevKitC-1-master/ESP32-S3-DevKitC-1主控.SchDoc` and MAX31850 datasheet:

| Component | Specification |
|-----------|---------------|
| MCU | ESP32-S3-DevKitC-1 @ 240MHz |
| Sensors | 4× MAX31850KATB+ (K-type thermocouple digitizer) |
| Bus Protocol | 1-Wire, 4 sensors paralleled on single bus |
| GPIO | IO14 (GPIO_NUM_14) |
| Pull-ups | R1=4.7KΩ per sensor to +3.3V |
| Addressing | Hardware via AD0/AD1 pins (00, 01, 10, 11) |

### MAX31850KATB+ Key Characteristics
- **Thermocouple Type**: K-Type only
- **Temperature Range**: -270°C to +1372°C
- **Resolution**: 0.25°C (14-bit thermocouple), 0.0625°C (12-bit cold junction)
- **Accuracy**: ±2°C (-200°C to +700°C)
- **Conversion Time**: ~100ms (auto-continuous, no command needed)
- **Family Code**: 0x3B
- **Data Frame**: 9-byte scratchpad with CRC8

### Technical Challenges
1. **1-Wire Timing Precision**: Requires microsecond-level timing accuracy
2. **Multi-device Management**: 4 sensors on single bus with Search ROM algorithm
3. **Signal Integrity**: 4.7KΩ pull-up may be too weak for 4-device bus
4. **CRC Validation**: X8+X5+X4+1 polynomial verification required

## 7. Problem Analysis

### Root Cause: Signal Integrity

The 4.7KΩ pull-up resistor creates a RC time constant with bus capacitance:

```
Rise time ≈ 2.2 × R × C

With 4.7KΩ and typical 100pF bus capacitance:
Rise time ≈ 2.2 × 4700 × 100e-12 = 1.03μs

With 4 devices (400pF):
Rise time ≈ 4.1μs (too slow for 1-Wire timing)

Recommended 2.2KΩ:
Rise time ≈ 1.9μs (acceptable)
```

### Impact on ROM Search

ROM Search requires reading 128 bits (64 bits × 2 reads) per device:
- Each bit read window: 15μs maximum
- Slow rise time causes sampling errors
- CRC detects bit errors but cannot correct them

## 8. Recommendations

### Option A: Hardware Modification (Recommended)

**Replace pull-up resistors**:
```
Current: 4.7KΩ
Recommended: 2.2KΩ (or even 1KΩ for better rise time)

Expected improvement:
- Rise time reduction from ~4μs to ~1.5μs
- Reliable ROM CRC validation
- Successful device enumeration
```

Test with single sensor to verify driver functionality:
```
Disconnect 3 sensors, test with only U1
```
