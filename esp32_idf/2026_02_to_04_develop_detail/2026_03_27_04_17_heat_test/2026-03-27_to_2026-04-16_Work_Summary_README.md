## 1. 项目概述

### 1.1 硬件配置

| 组件 | 规格 |
|------|------|
| 主控芯片 | ESP32-S3-DevKitC-1 (240MHz, 双核) |
| 温度传感器 | 4× MAX31850KATB+ (K型热电偶数字化器) |
| 总线协议 | 1-Wire，4个传感器并联在单一总线 |
| GPIO | IO14 (GPIO_NUM_14) |
| 上拉电阻 | 4.7KΩ (设计问题：应为每个传感器独立上拉) |
| 供电 | 3.3V外部供电 (非寄生模式) |

### 1.2 传感器特性

- **家族码**: 0x3B
- **温度范围**: -270°C 至 +1372°C
- **分辨率**: 0.25°C (14位热电偶), 0.0625°C (12位冷端)
- **转换时间**: ~100ms (自动连续转换)
- **数据格式**: 9字节Scratchpad + CRC8校验

### 3.2 第二阶段：问题排查 (03-28 至 03-31)

#### 发现的问题
| 问题 | 现象 | 分析 |
|------|------|------|
| 首次读取成功，后续失败 | 温度读取1次后CRC错误 | 时序漂移 |
| 字节顺序损坏 | `[XX 00 ...]` 而非 `[00 XX ...]` | 位级采样错误 |
| ROM搜索失败 | CRC校验失败，家族码0x3B读成0xDC | 位反转问题 |
| 重复设备ID | 搜索发现重复ROM ID | 总线信号完整性问题 |

#### 尝试的软件修复
1. ✅ 互斥锁保护整个操作
2. ✅ 关键区保护微秒级时序
3. ✅ 保守时序 (2-3倍标准值)
4. ✅ 读取重试机制 (最多3次)
5. ✅ 位级和字节级时序延迟

**结论**: 所有软件修复均无效，问题根源在硬件

### 3.3 第三阶段：硬件问题确认 (04-01 至 04-08)

#### 关键发现

**GPIO14开漏测试**
```
Drive LOW: 0 (expected 0)          ✅ 正常
Release (pull-up): 0 (expected 1)  ❌ 异常
Open-drain test: FAIL
```

**位反转分析**
```
读取的家族码: 0xDC (11011100)
期望的家族码: 0x3B (00111011)
关系: 位反转 (Bit Reversal)
```

#### 时序优化测试

| 方案 | READ_LOW | READ_SAMPLE | 采样点 | 结果 |
|------|----------|-------------|--------|------|
| 初始 | 6μs | 10μs | 16μs | ❌ CRC失败 |
| 方案1 | 3μs | 4μs | 7μs | ❌ CRC失败 |
| 方案2 | 2μs | 8μs | 10μs | ❓ 未完整测试 |
| 方案B | 1μs | 12μs | 13μs | ❌ CRC失败 |
| 诊断测试 | 1μs | 12μs+5μs | 18μs | ❌ CRC失败 |

**结论**: 位反转问题不是由信号上升沿时间导致，需要硬件修复

### 3.5 第五阶段：硬件沟通 (04-16)

#### 与Guoxiong MA沟通结果

**问题确认**:
- 每个MAX31850的DQ引脚应独立连接4.7KΩ上拉电阻
- 当前设计：4个传感器DQ并联后只使用1个4.7KΩ上拉
- 导致总线电容过大 (~200pF)，上升时间过长

**推荐的硬件修复方案**:
1. **短期**: 将4.7KΩ更换为2.2KΩ或1KΩ，加快上升沿
2. **中期**: 将4个传感器分为2组，每组使用独立GPIO
3. **长期**: 重新设计原理图，每个传感器独立上拉

## 5. 问题与解决方案

### 5.1 根本原因分析

```
┌─────────────────────────────────────────────┐
│           传感器发现失败                      │
└─────────────────────────────────────────────┘
                      ↓
        ┌─────────────┴─────────────┐
        ↓                           ↓
┌───────────────┐          ┌─────────────────┐
│  上拉电阻设计  │          │  总线电容过载    │
│  缺陷         │          │  (4设备并联)    │
│  (概率: 高)   │          │  (概率: 高)     │
└───────────────┘          └─────────────────┘
        ↓                           ↓
├─ 4个传感器共用1个上拉        ├─ 4×25pF = 100pF
├─ 应为每个传感器独立上拉      ├─ 线缆电容 ~100pF
└─ 4.7KΩ对4设备总线太弱       └─ 总电容 ~200pF
        ↓                           ↓
├─ 信号上升沿过慢              ├─ RC时间常数过大
├─ 采样时总线未稳定            └─ 上升时间 > 10μs
└─ 位反转/CRC失败              (正常应<1μs)
```

### 5.2 解决方案对比

| 方案 | 改动内容 | 效果 | 难度 | 成本 |
|------|----------|------|------|------|
| 1. 总线分组 | 4个传感器分2组，独立GPIO | 85% | 低 | 无 |
| 2. 总线驱动器 | 添加DS2482-100芯片 | 95% | 中 | ¥5-10 |
| 3. 减小上拉电阻 | 4.7KΩ → 1KΩ | 60% | 极低 | 无 |
| 4. 有源上拉 | 添加MOSFET快速上拉 | 90% | 高 | ¥2 |

### 5.3 推荐实施路径

- **短期 (已执行)**: 代码卷回原状态，等待硬件改进
- **中期 (计划中)**: 更换上拉电阻为2.2KΩ或1KΩ，重新测试
- **长期 (备选)**: 重新设计原理图，每个传感器独立上拉
